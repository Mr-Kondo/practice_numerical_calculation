# 計算力学の「流儀」を知るための2つの実験

コンピュータを使ってモノの壊れ方や水の流れをシミュレーションする「計算力学」の世界には、いくつかの異なる手法（流儀）が存在します。代表的なものに、有限差分法、有限要素法、有限体積法、境界要素法、そして個別要素法や粒子法などがあります。

「なぜこんなに種類があるのか？」と思うかもしれません。それは、コンピュータは現実世界のような「滑らかで連続した空間」をそのまま計算することができないからです。計算するためには、空間を細かく切り分けて（これを離散化と呼びます）、有限個の点の集まりとして扱う必要があります。

空間を「どうやって切り分けるか」、あるいは「そもそも切り分けるのか」。この根本的な思想の違いを理解するために、2つの仮想実験（ベンチマークテスト）を行ってみましょう。

## 実験1：「穴のあいた板」を引っ張る（静的な問題）

最初の実験は、真ん中に丸い穴があいた金属の板を、左右から強く引っ張るテストです。穴の周辺には力が集中しやすく（応力集中）、どこから亀裂が入りそうかを予測するのが目的です。この問題では、「空間をどう切り分けるか」の違いがはっきりと出ます。

### 1. 有限差分法（FDM）の戦い方：【方眼紙のマス目】
空間を縦横の規則正しい「方眼紙のマス目（グリッド）」で区切って計算します。計算のルールはシンプルで高速ですが、丸い穴のカーブを四角いマス目で表現しようとすると、ファミコンのドット絵のように「階段状」になってしまい、穴周辺の計算精度が落ちてしまう弱点があります。

### 2. 有限要素法（FEM）の戦い方：【自由自在なジグソーパズル】
空間を三角形や四角形のブロック（要素・メッシュと呼びます）に分割します。丸い穴の周辺はブロックを細かくし、遠くの何もない部分はブロックを大きくする、といった柔軟な切り方が可能です。現在のモノづくり（構造解析）において、最も頼りにされている万能型の優等生です。

### 3. 境界要素法（BEM）の戦い方：【輪郭だけをなぞる】
空間の中身を分割するのをやめ、板の「外枠」と穴の「縁（境界）」だけを細かく分割して計算します。計算するデータ量が圧倒的に少なく、かつ穴の周辺の精度が極めて高いという魔法のような手法です。ただし、後述するような「中身がダイナミックに壊れる」現象は計算できません。

### 【実験1の結論】
モノが大きく変形しない静かな問題では、空間をブロックに切り分ける「有限要素法（FEM）」や、輪郭だけを解く「境界要素法（BEM）」が圧倒的に有利です。では、個別要素法や粒子法はどうでしょうか？ 実は彼らは、こうしたカッチリした静的な問題を解くのはとても苦手です。その理由は、次の実験で明らかになります。

### 境界条件と荷重の与え方（実験1詳細）

#### 問題設定

| パラメータ | 値 |
|---|---|
| 板の幅 W | 2.0 m |
| 板の高さ H | 1.0 m |
| 穴の半径 a | 0.15 m |
| 遠端応力 σ_∞ | 1.0（正規化） |
| 原点 | 板・穴の中心 |

板は x ∈ [−W/2, +W/2]、y ∈ [−H/2, +H/2] の矩形領域に置かれ、
中心に半径 a の円孔があります。

#### 境界条件

| 境界 | 条件 |
|---|---|
| 右端（x = +W/2） | 法線応力 σ_xx = σ_∞（引張荷重） |
| 左端（x = −W/2） | σ_xx = σ_∞、x 方向変位を対称固定 |
| 上下端（y = ±H/2） | 自由端（σ_yy = τ_xy = 0） |
| 穴面（r = a） | 自由面（σ_rr = σ_rθ = 0） |

荷重はスカラー「荷重係数」によって 0 → σ_∞ まで線形にランプアップされます。
アニメーションの各フレームはこの荷重増加の一段階を示します。

#### Kirsch の解析解

単軸引張 σ_∞（x 方向）下の無限板中の円孔まわりの周方向応力 σ_θθ は、
Kirsch（1898）の解析解で正確に与えられます。

極座標 (r, θ) — θ は x 軸から反時計回り — において：

$$\sigma_{\theta\theta}(r, \theta) = \frac{\sigma_\infty}{2}\left[\left(1 + \frac{a^2}{r^2}\right) - \left(1 + \frac{3a^4}{r^4}\right)\cos 2\theta\right]$$

穴面（r = a）では上式は次のように簡略化されます：

$$\sigma_{\theta\theta}(a, \theta) = \sigma_\infty\left(1 - 2\cos 2\theta\right)$$

θ = π/2（穴の上端・下端）で最大値となり、**応力集中係数 Kt = 3** を与えます：

$$\sigma_{\theta\theta}^{\max} = 3\,\sigma_\infty \quad (\text{at } r = a,\;\theta = \pi/2)$$

θ = 0（穴の左右端）では σ_θθ = −σ_∞（圧縮）となります。

#### 各手法の穴境界の表現方法

| 手法 | 穴境界の近似 | Kt 精度への影響 |
|---|---|---|
| FDM | 矩形格子セルの中心で内外を判定 → 階段状（stair-step）境界 | 最近傍格子点が穴面より離れるため Kt を過小評価 |
| FEM | 穴面付近にメッシュを細化した滑らかな円境界 | 穴面近傍の数値積分誤差は小さく Kt ≈ 3 |
| BEM | 穴の縁（境界）のみを直接離散化 → 解析的に正確な円境界 | Kt = 3 を厳密に再現 |

## 実験2：津波が柱に激突して砕け散る（動的な問題）

次の実験は、せき止めていた大量の水が一気に崩れ落ち（ダムブレイク）、行く手にあるゴムのような弾力のある柱に激突するテストです。柱は極限まで曲がり、やがて千切れて水しぶきと共に飛散します。

この実験では、空間をブロック（メッシュ）で切り分ける手法の「限界」が露呈します。

### 1.有限要素法（FEM）の限界：【伸びきったパズル】
水がぶつかり柱が大きく曲がるところまでは計算できます。しかし、柱が「千切れる」、あるいは水が「水しぶきになって散る」瞬間、空間を分割していた三角形や四角形のブロックがゴムのように異常に引き伸ばされ、計算機の中でエラーを起こして破綻してしまいます（メッシュの崩壊）。

### 2.有限体積法（FVM）の戦い方：【箱の中の出入りを監視】
流体の計算を大得意とする手法です。空間に固定の「箱（コントロールボリューム）」を置き、その箱を水や空気がどれだけ出入りしたかを厳密に計算します。水流の計算には強いですが、飛散する水しぶきの「表面」を正確に追いかけるのには特殊な工夫が必要です。

### 3. 個別要素法（DEM）・粒子法（SPHなど）の戦い方：【砂つぶと水滴】
ここで真打ちが登場します。これらの手法は、そもそも空間をブロックで切り分けること（メッシュ）を放棄しています。モノや水を、無数の「点（粒子）」の集まりとして扱います。
柱は「バネで繋がれた砂つぶの集まり」、水は「自由に動く水滴の集まり」として計算します。そのため、柱が千切れるときは単に「バネが切れた」として扱い、水しぶきは「粒子がバラバラに飛んでいった」として、ごく自然に計算できます。どんなにメチャクチャに壊れても計算が破綻しません。

### 【実験2の結論】
モノが千切れる、飛散する、混ざり合うといった激しい現象（トポロジー変化）が起きる領域では、メッシュを必要としない「粒子」ベースの手法が絶対的な威力を発揮します。

## まとめ：万能な手法は存在しない

**形が保たれる問題は、有限要素法（FEM）などの「メッシュで切る」手法が高精度で有利。**

**形が壊れる・散る問題は、粒子法や個別要素法（DEM）などの「粒子で動かす」手法が有利。**

--- 

エンジニアの仕事は、「どの手法が一番優れているか」を決めることではありません。自分がこれから解きたい問題（現象）の性質を見極め、最も適切な「道具（計算手法）」をツールボックスから選び出すことです。これが計算力学の第一歩となります。

## 出力図の見方

各ベンチマークの実行後に生成されるアニメーション（GIF/MP4）と静止画フレーム（PNG）について、各パネルが示す物理量を以下に説明します。

### 実験1（B01）：穴のあいた板の引張

アニメーションは **1 行 3 列のパネル**（左: FDM、中: FEM、右: BEM）を並べ、
荷重係数を 0.05 → 1.0 まで線形にランプアップしながら各手法の 2 次元応力場を
時系列で表示します。

各パネルの見方：
- **カラーマップ（coolwarm）**: 青が圧縮（σ_θθ < 0）、赤が引張（σ_θθ > 0）。
	スケールは荷重係数に比例し、最終フレームでは −σ_∞（青）〜 3σ_∞（赤）の範囲に対応します。
- **白い円**: 穴の内部（計算対象外）。
	FDM パネルでは穴の輪郭が**階段状**になっているのが確認できます（格子セルによる近似）。
	FEM・BEM パネルでは滑らかな円形境界になっています。
- **矢印（→ ←）**: 板の左右端に加えられた一様引張荷重 σ_∞ の方向を示します。
- **応力集中**: 穴の上端・下端（y 方向）に赤いホットスポットが現れ、
	最終フレームで σ_θθ ≈ 3σ_∞（Kt = 3）が確認できます。
- **FEM の数値ノイズ**: FEM パネルでは穴近傍に小さなランダムな濃淡が見られます。
	これは有限要素の数値積分誤差を模擬しています。

### 実験2（B02）：ダムブレイク崩壊

アニメーションは1つの画面に3つのパネルを並べ、それぞれ異なる数値手法によるダムブレイクの時間発展を表示します。

**左：FVM free-surface section（有限体積法）**

ここはFVMの1次元浅水計算で得られる水深 $h(x)$ を、**2次元互換の占有セル場**として再構成して表示したパネルです。横軸は正規化座標 x/L、縦軸は正規化高さ y/H で、青いセルほどその高さ帯が液相で満たされていることを表します。上端の線は同じ時刻の自由表面高さです。

この表示は2次元VOF/Level Setを直接解いたものではなく、1次元水深分布を2次元セル場へ押し出して再構成したものです。したがってSPHの粒子配置と同じ意味ではありませんが、少なくとも「2次元格子上のどこが液体か」という出力契約を先に揃えた形になっています。FVMでは引き続き「前線位置・濡れ幅・残留水深」の読み取りに主眼を置いてください。

初期状態では左25%（x/L≈0.0〜0.25）が高水位、右側はほぼ乾燥です。時間発展では、
- 左側で水面がなだらかに低下する（疎波）
- 右方向に前線が進行する（ボア）
- 開放右境界から流出し、終盤は計算領域内の残留水が減る

という1次元ダムブレイクの特徴を確認できます。

**右：FEM collapse geometry（有限要素法）**

ここではFEMの節点を散布図で表示し、柱がつぶれて崩れる様子を直接見せます。右パネル内の小さなインセットには品質指標が入っています。

見るポイントは次の2つです。
- 柱の形が縦長から横に崩れていくか。
- 品質指標（min_quality）が下がり、collapse比率が上がるか。

つまり「形の崩れ」と「数値品質の悪化」を同時に読むパネルです。

このパネルはSPHと**同じ固定表示範囲**で描いています。したがって、FEMだけが常に画面いっぱいに見える問題を避け、3手法の見た目を同じ物差しで比較できます。

**中央：SPH particles（粒子法）**

粒子法（SPH代理ソルバー）による流体粒子の空間分布を示します。各点が1つの流体粒子を表します。横軸は粒子位置 x/L、縦軸は粒子位置 y/H です。初期状態では矩形のダム形状（x∈[0, 0.25]、y∈[0, 0.8]）に粒子が密集しています。重力によって粒子群が下方へ加速し、床（y=0）に衝突した後は主に右方向へ拡散していく様子を確認できます。メッシュを持たないため、粒子がどれほど飛散しても計算が破綻しません。

#### B02でFVMとSPHの軸範囲が異なる理由

- **X軸の範囲が異なる理由**
	- FVMは1次元の中心線場を扱うため、表示範囲は定義域の x/L∈[0,1] です。
	- SPHは粒子の実座標をそのまま描画します。現在の設定では左壁と床のみを反射境界とし、右壁を設けていないため、粒子は右方向へ自由に広がり x/L>1 に到達します。
	- FVMも右境界は開放に変更しましたが、定義域そのものは x/L∈[0,1] に固定されたままです。したがって、FVMでは「右へ流れ去る」ことは表現できても、x/L>1 の外側まで形状を描くことはできません。

- **Y軸の最高値が異なる理由**
	- FVMのY軸は「水深 h/h0」という場の量で、初期最大値は1.0です。
	- SPHのY軸は「粒子の位置 y/H」であり、初期の粒子配置上限が約0.8なので、上限はFVMの1.0とは一致しません。

要するに、FVMとSPHは同じ「高さ」や「位置」を描いているのではなく、**異なる物理量**を表示しています。今回のFVMは2次元互換セル場を出力するようにしたものの、物理モデル自体はまだ1次元浅水です。したがって、軸上限の一致を期待するより、各手法が何を示しているかを軸ラベルで読み分けるのが正しい解釈です。なお、開放右境界にしたFVMでは質量の一部が領域外へ流出するため、`mass_error` は閉領域の保存誤差ではなく「初期質量からの減少量」として読む必要があります。

比較条件はSPHに合わせ、壁条件（left wall + floor、right wallなし）を統一しています。さらに、FEM/SPHの主表示は同じ固定表示範囲で描画し、見た目の比較の公平性を保っています。

現在のB02出力では、FVM/FEM/SPHのすべてが `viz_timeseries.frame_times` を持つようにしてあり、各フレームが何秒時点かを共通の尺度で追えるようにしています。ただし、FVMだけは内部CFL時刻を使うため、同じフレーム番号でも他手法の `dt` ベース時刻と完全一致するわけではありません。公平比較をさらに進めるには、次段階として「共通の比較時刻へ補間してから可視化する」必要があります。

## ベンチマークの実行方法

この実験を再現するには、まず依存関係を同期します。通常は `uv sync` を実行し、GPU版（PyTorch）も使う場合は `uv sync --extra gpu` を選びます。

2つの実験をまとめて実行する場合は、プロジェクトルートで `uv run sec01-run-all` を実行します。
個別に実行する場合は、実験1が `uv run sec01-b01`、実験2が `uv run sec01-b02` です。

実行結果は、それぞれ `sec_01/outputs/B01` と `sec_01/outputs/B02` に保存されます。
